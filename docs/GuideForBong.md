# GuideForBong — 이 프로젝트를 제대로 이해하기

> "이 프로젝트를 하면 RAG·AI Agent·LLM Wiki가 실제로 적용되는 건가? 내가 뭘 배우고, 실제로 어떤 기술이 쓰이나?" 에 대한 자세한 답.

---

## 1. 먼저 오해부터 풀기 — "두 개의 AI"

이 프로젝트를 이해하려면 **완전히 다른 두 AI**를 구분해야 합니다.

| | **Claude Code (나)** | **ROS2 Copilot (당신이 만드는 것)** |
|---|---|---|
| 정체 | 당신 옆에서 코드를 짜주는 보조 AI | 로봇 안에서 도는 **당신의 애플리케이션** |
| 언제 동작? | 지금, 개발할 때 | 런타임, 로봇이 돌 때 |
| 누가 만들었나 | Anthropic | **당신** |
| RAG/Agent/Wiki는? | 내가 내부적으로 쓸 수도 있지만 그건 이 프로젝트와 무관 | **당신이 코드로 직접 구현** |

당신이 헷갈린 지점: *"RAG/Wiki는 AI가 성능 유지하려고 따르는 약속 아니야?"* — 그건 왼쪽(나)에 대한 오해예요.

**이 프로젝트는 오른쪽을 만드는 겁니다.** RAG·Agent·Wiki는 "약속"이 아니라 **당신이 짜는 코드**입니다. 벡터DB에 문서를 넣는 코드, LLM에 도구를 물려 루프를 도는 코드, 그래프를 읽어 문서를 뽑는 코드 — 전부 당신 손으로 구현합니다.

> 비유: 나는 "레시피를 알려주는 셰프"이고, ROS2 Copilot은 "당신이 직접 차리는 레스토랑"입니다. 이 프로젝트의 목표는 레스토랑을 차리는 것 — 요리(RAG/Agent/Wiki)는 당신이 배워서 직접 합니다.

---

## 2. 이 프로젝트에 정말 세 기술이 쓰이나? — 코드 위치로 확인

**네. 그리고 각각이 코드의 어디에 사는지 정확히 짚어줄게요.**

### 🟦 RAG — 실제로 쓰이는 곳
- **패키지**: `copilot_rag` (Python/rclpy)
- **당신이 짜는 코드**:
  - 문서(ROS2 매뉴얼, 소스, 설정)를 청킹하는 함수
  - 임베딩 → 벡터DB(Chroma/pgvector)에 저장하는 파이프라인
  - 질문이 오면 유사 문서를 검색 → 프롬프트 조립 → Claude 호출 → 출처와 함께 답변
- **이게 RAG인 이유**: LLM이 자기 기억이 아니라 **당신이 검색해서 준 문서**에 근거해 답함. 그 "검색해서 주는" 부분이 당신 코드.

### 🟩 AI Agent — 실제로 쓰이는 곳
- **패키지**: `copilot_agent` (Python 두뇌) + `copilot_executor` (C++ 실행)
- **당신이 짜는 코드**:
  - `navigate_to`, `get_robot_state`, `query_knowledge` 같은 **도구(함수)를 정의**
  - Claude에게 도구 목록을 주고, Claude가 "어떤 도구를 어떤 인자로 부를지" 결정하면 **당신 코드가 그 도구를 실제로 실행**하고 결과를 다시 Claude에 돌려주는 **루프**
- **이게 Agent인 이유**: LLM이 한 번 답하고 끝이 아니라, 도구를 골라 쓰고 결과를 보고 다음을 판단하는 **여러 단계 자율 실행**. 그 루프와 도구가 당신 코드.

### 🟨 LLM Wiki — 실제로 쓰이는 곳
- **패키지**: `copilot_wiki` (Python)
- **당신이 짜는 코드**:
  - 실행 중인 ROS2 그래프를 introspection (`ros2 node/topic list`)
  - 그 사실 데이터를 Claude에 주고 **구조화된 문서(마크다운 위키)를 생성**하도록 요청
  - mermaid 다이어그램 + 크로스링크 자동 생성
- **이게 Wiki인 이유**: LLM이 **당신 시스템의 실제 구조**를 읽어 사람이 읽을 문서로 정리. 그 introspection과 생성 트리거가 당신 코드.

## 3. 그래서 내가 배우는 게 정확히 뭔데?

**"AI를 쓰는 사람"이 아니라 "AI 시스템을 만드는 엔지니어"가 되는 것.** 구체적으로:

### AI/LLM 엔지니어링
- LLM API를 코드에서 호출하고 프롬프트를 설계하는 법
- RAG 파이프라인을 **직접 구축** (청킹·임베딩·검색·리랭킹·출처)
- 에이전트 루프와 **도구 설계** (tool calling, 멀티스텝 추론)
- 구조화 생성·grounding으로 할루시네이션을 억제하는 법
- LLM 시스템을 **평가**하는 법 (정확도·성공률·비용)

### 로봇 개발 (당신 채용 타겟의 핵심)
- **C++ (rclcpp)**: 실시간·안전 노드, 액션 서버 — 성능·시스템 프로그래밍 실력 증명
- **Python (rclpy)**: 서비스/노드, AI 오케스트레이션
- ROS2 아키텍처: 노드·토픽·서비스·액션·파라미터·tf2·QoS
- 커스텀 인터페이스 설계, colcon 멀티패키지 워크스페이스
- Gazebo·Nav2로 **실제 로봇 시뮬레이션** 구동

### 시스템 설계
- 왜 레이어를 나누고 ROS2 인터페이스로 통신하는가
- "LLM이 틀려도 로봇이 안전한" 안전 계층 설계 (면접에서 강력한 포인트)

**한 줄 요약**: 뻔한 "ChatGPT 래퍼"도, 뻔한 ROS2 튜토리얼도 아닌 — **둘을 결합해 직접 만든 시스템**. 이게 당신을 다른 지원자와 구분해줍니다.

## 4. 실제로 쓰이는 기술 스택 (구체적)

| 분류 | 기술 | 이 프로젝트에서의 역할 |
|---|---|---|
| **로봇** | ROS2 (Humble/Jazzy) | 전체 뼈대 — 노드/토픽/서비스/액션 |
| | Nav2 | 자율주행 스택 (Phase 4) |
| | Gazebo | 시뮬레이션 (Phase 4) |
| | tf2 | 좌표 변환 / 로봇 상태 |
| | colcon | 멀티패키지 빌드 |
| **언어** | C++17 (rclcpp) | 실시간·안전 노드 |
| | Python 3.10+ (rclpy) | AI 레이어·글루 |
| **LLM** | Claude API (`claude-sonnet-5` / `claude-opus-4-8`) | 세 레이어의 두뇌 |
| | anthropic SDK | Claude 호출, tool calling |
| **RAG** | 임베딩 + 벡터DB (Chroma → pgvector) | 문서 검색 |
| **개발** | git, Docker(선택) | 버전관리·환경격리 |

## 5. 런타임에 실제로 무슨 일이 일어나나 (예시 흐름)

사용자가 로봇에게 *"창고 구역이 좁아 보이는데, 거기로 가도 괜찮아? 괜찮으면 가"* 라고 말했을 때:

```
1. 사용자 발화 → copilot_agent (Python)
2. Claude가 판단: "먼저 지식을 확인해야겠다" → query_knowledge 도구 호출
3. copilot_rag가 costmap/inflation 관련 문서를 벡터DB에서 검색
   → Claude가 출처와 함께 "통로 폭이 로봇보다 크면 통과 가능" 답변  [🟦 RAG]
4. Claude가 다음 판단: "이동해야겠다" → navigate_to(창고 좌표) 도구 호출  [🟩 Agent]
5. copilot_executor (C++)가 목표를 받아 안전 검증(맵 범위·속도·e-stop)
   → 통과하면 Nav2에 이동 명령 위임
6. C++ Safety Monitor가 /scan을 보며 장애물 감지 시 긴급 정지 (LLM보다 우선)
7. 이동 완료 → Claude가 결과를 자연어로 사용자에게 보고
```

여기서 **2·4단계의 판단은 Claude가**, **3·5·6단계의 실행·검색·안전은 당신이 짠 코드가** 합니다. 이게 "AI Agent + RAG를 실제로 적용한다"의 의미예요.

그리고 나중에 개발자가 *"이 로봇 시스템 문서 만들어줘"* 하면 `copilot_wiki`가 그래프를 읽어 노드별 위키를 자동 생성합니다. [🟨 LLM Wiki]

## 6. 정리

- 이 프로젝트는 **당신이 RAG·Agent·Wiki를 직접 구현**하는 프로젝트입니다. (내가 대신 하는 게 아니라)
- 세 기술은 각각 `copilot_rag` / `copilot_agent`+`copilot_executor` / `copilot_wiki` 패키지로 **실제 코드에 존재**합니다.
- 배우는 것: **LLM 시스템 엔지니어링 + ROS2 로봇 개발(C++/Python) + 시스템 설계.**
- 결과물: 돌아가는 로봇 데모 + 직접 만든 AI 시스템 = **차별화된 개발자 포트폴리오.**

이제 [로드맵](ROADMAP.md)의 Phase 0부터 실제로 코드를 만들어 나갑니다. 궁금한 게 생기면 이 문서를 다시 보세요.
